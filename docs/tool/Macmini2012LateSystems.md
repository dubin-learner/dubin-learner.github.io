# Mac mini 2012 Late安装三系统

## 写在前面

大概是2021年，我在闲鱼平台上收了一台Mac mini 2012 Late，CPU是英特尔i5的低压U，替代了从本科用到工作的Dell N4050笔记本。其实更换固态硬盘之后，笔记本使用还算流畅，只不过发热比较严重；而且又厚又重，比较占地方。考虑到我只在出租屋用，并且当时已经有了一个1080p的AOC外接显示器，加上想体验一下macOS，于是买了二手。

这款Mac mini 2012 Late比起后面更新的版本，最大的优点就是内存和硬盘都可定制，内存可以使用笔记本内存，增大到16G；硬盘支持安装上下两块。即使CPU性能一般，日常使用也基本无压力。后来发现DP口居然支持外接4K显示器，官方的技术规格似乎写的是2K，确实是意外之喜。

所以这台小主机到手之后，没少被我折腾。加了内存和固态硬盘；最开始用macOS Catalina，官方只给支持到这个版本。然后尝试用Bootcamp安装Windows10，一度让Win10成了使用主力，甚至在居家办公时期成了我的主力工作电脑。（后来前公司VPN彻底不开放，自然只能在公司才能工作。问原因的话就是防患于未然，怕自己那点见不得人的商业秘密泄漏）。再后来为了装一些只有Linux环境才能跑的软件，我还加装了Ubuntu20.04。小小主机装了三个系统，也算是挖掘潜能了。

当然装第三个系统的时候还是有些波折的。当时只有一块固态硬盘，装上Ubuntu后Windows立刻就出了问题，有启动项但进去就崩溃，找不到Windows系统。后来查了一下，原来是和启动项管理有关，一块盘上能写的引导项有限制，装不了三个系统。于是又增加了一块固态硬盘，原来的那块还是装macOS和Windows保持不变，新的盘只装Ubuntu。折腾了几次之后终于点亮，每个系统都互不影响，还是有点开心的。

说了这么多，本文主要记录一下装系统过程中遇到的问题。现在我已经不太愿意折腾，小主机的无线模块也出了一点问题，三个系统都连不上WiFi。大概以后也很少会再用了吧，不过我还是会尽量好好保养它的。这些问题距今已经有段时间，我准备借助DeepSeek进行快速总结，当然参考文章会在最后列出。

## 一、Ubuntu无线网卡驱动安装实战

这个比较麻烦，麻烦的点在于我没有有线网络。理论上，只有有根网线，这个问题就不难了；但出租屋的环境并不支持，路由器在公共区域。

我后来摸索出一个方法，就是利用手机共享网络。无论是安卓还是苹果都可以，利用USB连接电脑，打开共享网络，苹果的话用个人热点就行。这样直连之后，ubuntu会将这个USB连接识别位有线网络，这样就能进行无线模块的安装。

最直接的方法：
```bash
sudo apt-get update
sudo apt-get install bcmwl-kernel-source
```

通常这样就可以了。如果遇到其他问题，继续看参考文章吧。

参考文章：[难题 [已解决] ：如何给Ubuntu系统安装无线网卡驱动？](https://blog.csdn.net/weixin_39278265/article/details/86604677)

## 二、Ubuntu键盘布局修正

在安装Ubuntu过程中，如果误选了键盘布局（例如选择了英国英语，而非美国英语），会导致某些按键输出错误字符（如按 `#` 键输出 `£`）。修正方法如下：

1. 打开终端，输入命令进入配置界面：
    ```bash
    sudo dpkg-reconfigure keyboard-configuration
    ```
2. 跟随向导进行选择：
    *   **键盘型号**：选择“通用105键（国际）电脑”。
    *   **键盘布局国家**：选择“美国”。
    *   **键盘默认布局**：保持默认。
    *   **组合键**：选择“无组合键”。
3. 配置完成后，检查系统右上角的输入法/键盘图标，确保其显示为“英语（美国）”。此方法可快速解决因布局错误导致的基础符号输入问题。

参考文章：[Ubuntu 如何设置成中国人默认使用的键盘布局](https://zhuanlan.zhihu.com/p/1895735606946017323)

## 三、Mac启动项中的“EFI Boot”清理
在Mac上通过Boot Camp安装并卸载Windows后，启动时按住`Option`键，可能会看到一个名为“EFI Boot”的多余启动图标。这是Windows的启动文件残留在EFI系统分区中所致。

### 1. 什么是EFI Boot？
“EFI Boot”是一个启动入口，当Mac的固件在内部磁盘的EFI分区中的 `\EFI\BOOT\BOOTx64.EFI` 路径下检测到启动文件时，便会显示此图标。这通常发生在安装Windows之后。

### 2. 通过终端删除残留EFI启动项
**警告**：操作EFI分区具有一定风险，请务必精确按照步骤执行，避免误删其他关键文件。
1. 打开 **终端**。
2. 列出所有磁盘，找到EFI分区（通常类型名为 `EFI`，位置例如 `/dev/disk0s1`）：
    ```bash
    diskutil list
    ```
3. 手动挂载EFI分区（将 `disk0s1` 替换为你找到的实际位置）：
    ```bash
    sudo mkdir /Volumes/EFI
    sudo mount -t msdos /dev/disk0s1 /Volumes/EFI
    ```
4. 进入EFI目录，查看文件夹内容：
    ```bash
    cd /Volumes/EFI/EFI
    ls
    ```
    此时应能看到一个名为 `Microsoft` 的文件夹，这即是Windows的启动残留。
5. **删除整个Microsoft文件夹**：
    ```bash
    sudo rm -rf Microsoft
    ```
6. 操作完成后，重启Mac并按住`Option`键，“EFI Boot”图标应不再出现。
