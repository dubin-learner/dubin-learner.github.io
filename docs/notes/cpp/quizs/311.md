# C++ Quiz: Question 311

[原文地址](https://cppquiz.org/quiz/question/311)

---

**问题 #311** 难度：1

根据C++23标准，这个程序的输出是什么？

```cpp
#include <iostream>
#include <typeinfo>

struct Base
{
};

struct Derived : Base
{
};

int main()
{
    Derived d;
    Base& b = d;
    std::cout << (typeid(b) == typeid(Derived));
}
```

## 答案

<details>
<summary>内容展开</summary>

程序输出：
```cpp
0
```

**正确！**

`b` 是一个`Base`引用，指向一个类型为`Derived`的对象。但由于`Base`没有虚函数，它没有运行时类型信息，`typeid(b)`返回的是类型`Base`而不是`Derived`。比较结果为`false`，因此打印出`0`。

**细节：**

首先，我们可以忽略`b`是一个引用这一事实。`typeid`将返回被引用类型的类型信息。§[expr.typeid]¶5：

> 如果`type-id`的类型是可能带有cv限定符的类型的引用，则`typeid`表达式的结果指向一个表示该cv无限定引用类型的`std::type_info`对象。

但`typeid(b)`的结果是什么？是它的静态类型`Base`，还是它的动态类型`Derived`？§[expr.typeid]¶4：

> 当`typeid`应用于非多态类类型（glvalue）的表达式时，结果指向一个表示该表达式静态类型的`std::type_info`对象。

`Base`是多态类类型吗？不是。§[class.virtual]¶1：

> 包含虚成员函数的类称为多态类。

由于`Base`中没有虚函数，它不算作多态类。因此`typeid`返回的是静态类型`Base`，而不是动态类型`Derived`。

这种不便之处背后的原因是“不为不用的东西付费”原则。C++类实例在不需要的情况下不会携带任何类型信息。如果类有虚函数，则需要一些类型信息（通常以虚表指针的形式）来多态地选择正确的派生函数。
